## Лабораторная работа 10
### Краткая теория
#### ➯  Стек (Stack)
Это структура данных, работающая по принципу LIFO (Last In, First Out — «последним пришёл, первым ушёл»).
#### ➯ Очередь (Queue)
Работает по принципу FIFO (First In, First Out — «первым пришёл, первым ушёл»).
#### ➯ Односвязный список (Singly Linked List)
Это динамическая структура данных, состоящая из узлов. 
Каждый узел содержит само значение и ссылку (указатель) на следующий узел.
В отличие от массива, элементы списка могут быть разбросаны в памяти.



### Классы и их операции 
#### ➯ Класс Stack
Реализован на базе стандартного list в Python.
- push(item): Добавляет элемент в конец массива через append().
- pop(): Извлекает элемент с конца. Реализована обработка исключения IndexError для предотвращения падения программы при пустом стеке.
- peek(): Позволяет посмотреть верхний элемент без удаления, используя индекс [-1]. 
#### ➯ Класс Queue

Реализован с использованием collections.deque для оптимизации работы с началом очереди.

- enqueue(item): Добавляет элемент в конец двусторонней очереди.
- dequeue(): Использует метод popleft() для удаления первого элемента за константное время.
- is_empty(): Проверяет наличие элементов в очереди.
#### ➯ Класс SinglyLinkedList
Реализация на базе узлов Node с двумя указателями: head и tail.
- Управление хвостом: Наличие атрибута self.tail позволяет добавлять элементы в конец за O(1), не проходя через весь список.
- Вставка/Удаление: Методы insert и remove_at корректно перестраивают связи (указатели next) и обновляют размер списка self._size.
- Итерация: Реализован метод __iter__ с использованием ключевого слова yield, что позволяет обходить список циклом for.
